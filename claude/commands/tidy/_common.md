# Tidy 공통 규칙

모든 `/tidy:*` 명령어에 적용되는 공통 규칙입니다.

## 핵심 원칙

> **필요 최소한의 정리만 수행. 과도한 정리로 기존 내용을 생략하지 않는다.**

## 필수 준수 사항

### 1. 최소한의 정리만 수행

- 명백한 중복/오류만 수정
- 과도한 정리로 기존 내용 생략 금지
- 의심스러우면 그대로 유지
- "정리"가 아닌 "개선"은 하지 않음

### 2. 원본 내용 보존

- 의미가 달라지는 변경 금지
- 의도적 반복(강조)은 유지
- 예시, 참고, 주석 내용 삭제 금지
- 코드 로직 변경 금지 (포맷팅만 허용)

### 3. 최근 수정 파일 건너뛰기

- 파일의 마지막 수정 시간 확인
- **24시간 이내 수정된 파일은 건너뜀**
- `[스킵] {파일명} - 최근 정리됨` 출력

### 4. 대폭 변경 시 확인

- **20% 이상 줄어드는 경우** 변경 내역 먼저 보여주기
- 사용자 승인 후 저장
- 불확실한 경우 항상 사용자에게 확인

## 출력 형식

```
[대상] N개 파일 발견
[스킵] file1.md - 최근 정리됨
[분석] file2.md (100줄)
[정리] 완료 (100줄 → 95줄, -5%)
[저장] file2.md ✓

[요약] N개 파일 처리, M개 스킵, 총 X줄 감소
```
