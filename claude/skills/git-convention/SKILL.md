# Git 컨벤션

## 커밋 메시지 형식
```
<타입>: <제목>

<본문 (선택)>
```

## 타입
- feat: 새 기능
- fix: 버그 수정
- docs: 문서 수정
- style: 포맷팅 (코드 변경 없음)
- refactor: 리팩토링
- test: 테스트 추가/수정
- chore: 빌드, 설정 등

## 규칙
- 제목은 50자 이내
- 제목 끝에 마침표 없음
- 본문은 72자에서 줄바꿈
- 한국어로 작성

## 브랜치 명명
- feature/기능명
- fix/버그명
- docs/문서명

## 예시
```
feat: 로그인 페이지 추가

- 이메일/비밀번호 폼 구현
- 유효성 검사 추가
```
