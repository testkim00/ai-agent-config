# 실행 결과 보고서 템플릿

오케스트레이션 완료 후 콘솔에 출력하는 결과 보고서 템플릿입니다.

## 용도

- 작업 완료 후 사용자에게 결과 요약 제공
- 변경 파일 목록 및 상세 설명
- 개선 효과 및 아키텍처 변경 사항

## 섹션별 설명

### ■ 요약
- **작업**: 수행한 작업의 한 줄 요약
- **총 실행 시간**: 1~4단계 총 소요 시간 (초)
- **생성/수정/삭제 파일**: 변경된 파일 수

### ■ 완료 작업 목록
- TodoWrite에서 관리한 작업 목록
- 상태: ✅ (완료), ⏳ (진행중), ❌ (실패)

### ■ 변경 파일 상세
- 상태: `신규`, `수정`, `삭제`
- 각 파일에 대한 간단한 설명

### ■ 단계별 소요 시간 ⭐ 필수
- 1~4단계별 소요 시간과 비율
- 판정 기준:
  - 🔴 심각한 병목: 50% 이상
  - 🟡 주의 필요: 30~50%
  - 🟢 정상: 30% 미만

### ■ 병목 분석 ⭐ 필수
- 가장 오래 걸린 단계 식별
- 원인 분석 (직접 분석 또는 담당자 인터뷰)
- 개선 제안

### ■ 병렬 실행 효율성 ⭐ 필수
- 병렬 효율성 = (순차 예상 시간) / (실제 시간)
- 판정 기준:
  - 🟢 우수: 2.0x 이상
  - 🟡 양호: 1.5x ~ 2.0x
  - 🔴 미흡: 1.5x 미만
- 병렬 그룹별 시간 편차 분석
- 병렬화 기회 손실 식별

### ■ 프로세스 개선 제안 ⭐ 필수
- Orchestrator와 논의 결과
- 계획/실행 전반에 대한 검토
- 개선방안 분류:
  - 즉시 적용: 명확한 효과, 부작용 없음
  - 검토 필요: 효과 예상되나 검증 필요
  - 장기 과제: 구조적 변경 필요

### ■ 개선 효과
- 카테고리별 개선 내용 정리
- 예: 중복 코드 제거, 성능 개선, 코드 품질 향상

### ■ 새 아키텍처
- 구조 변경이 있는 경우에만 포함
- 계층별 모듈 구성 설명

## 작업 유형별 템플릿 변형

### 리팩토링 작업
```
■ 개선 효과
┌─────────────────────────────────────────────────────────────┐
│ 1. 중복 코드 제거                                            │
│    - {before} → {after}                                      │
│                                                              │
│ 2. 성능 개선                                                 │
│    - {optimization_detail}                                   │
│    - 예상 속도 향상: {X}배                                   │
│                                                              │
│ 3. 코드 품질 향상                                            │
│    - {quality_improvement}                                   │
└─────────────────────────────────────────────────────────────┘
```

### 신규 기능 개발
```
■ 구현 기능
┌─────────────────────────────────────────────────────────────┐
│ 1. {feature_1}                                               │
│    - 엔드포인트: {endpoint}                                  │
│    - 기능: {description}                                     │
│                                                              │
│ 2. {feature_2}                                               │
│    - 컴포넌트: {component}                                   │
│    - 기능: {description}                                     │
└─────────────────────────────────────────────────────────────┘
```

### 버그 수정
```
■ 수정 내용
┌─────────────────────────────────────────────────────────────┐
│ 원인: {root_cause}                                           │
│ 해결: {solution}                                             │
│ 영향 범위: {affected_scope}                                  │
└─────────────────────────────────────────────────────────────┘
```

## 추가 개선 제안 섹션 (선택적)

작업 완료 후 추가로 개선할 수 있는 항목이 있으면 포함:

```
■ 추가 개선 제안
┌──────────┬─────────────────────────────────────────────────┐
│ 우선순위 │ 항목                                            │
├──────────┼─────────────────────────────────────────────────┤
│ 높음     │ {suggestion_1}                                  │
│ 중간     │ {suggestion_2}                                  │
│ 낮음     │ {suggestion_3}                                  │
└──────────┴─────────────────────────────────────────────────┘
```

## 사용 시점

1. 오케스트레이션 5단계 완료 후
2. 모든 작업 완료 시점에 콘솔 출력
3. 사용자가 결과를 한눈에 파악할 수 있도록 정리
