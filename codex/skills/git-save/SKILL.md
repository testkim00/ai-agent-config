# Git Save

## 언제 사용하나

- 현재 변경을 커밋으로 저장할 때
- 커밋 메시지를 정리해야 할 때

## source mapping

- Claude command: `/git:save`

## 기본 규칙

- 변경사항이 없으면 바로 종료한다.
- 먼저 `git diff --stat`와 필요 시 `git diff`를 확인한다.
- 커밋 메시지와 브랜치 규칙은 `git-convention`을 따른다.
- 기본은 관련 파일만 명시적으로 stage 한다.
- `git add -A`는 범위가 명확할 때만 사용한다.

## 실행 절차

1. `git status --short`와 최근 커밋을 확인한다.
2. 변경 범위를 검토한다.
3. 이번 요청 범위에 해당하는 파일만 stage 한다.
4. 커밋 메시지를 만든다.
5. `git commit -m ...`로 커밋한다.
6. 커밋 해시와 포함 파일을 요약한다.

## 사용자에게 물어봐야 하는 경우

- unrelated change가 함께 섞여 있을 때
- 커밋 제목 방향이 여러 개로 갈릴 수 있을 때

## 하지 말아야 할 것

- 변경 범위를 확인하지 않고 무조건 전체 stage
- 사용자 변경을 임의로 되돌린 뒤 커밋
- amend를 명시 요청 없이 수행
