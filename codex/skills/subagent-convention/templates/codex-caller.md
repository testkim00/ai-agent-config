# Codex Caller Template

이 파일명은 과거 이름을 유지하지만, 현재 Codex에서는 `Codex CLI를 다시 호출`하는 템플릿으로 쓰지 않는다.

핵심 원칙:

- Codex 안에서 다시 `codex exec`를 중첩 호출하지 않는다.
- 두 번째 에이전트가 필요하면 `spawn_agent`를 사용한다.
- secondary pass가 필요할 때는 `explorer` 또는 `default`에 짧은 검토 brief를 보낸다.

## 검토용 brief

```text
목표: {검토 또는 자문 목적}
대상:
- {파일/모듈/계획}
질문:
- {핵심 질문 1}
- {핵심 질문 2}
출력:
- 핵심 판단
- 리스크
- 추천 방향
수정 여부:
- read-only 또는 수정 허용 여부를 명시
```

## 언제 유용한가

- 계획 검토
- 리팩터링 위험 점검
- 특정 기술 선택지 비교

## 언제 불필요한가

- 메인 에이전트가 바로 판단 가능한 간단한 문제
- 단순 파일 수정
- 즉시 결과가 필요한 blocker
